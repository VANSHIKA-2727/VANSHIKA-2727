**Problem Solving Approach**

Problems are approached by first identifying the optimal substructure, then verifying edge cases before writing a single line of code. Primary language is **C++** for its performance characteristics in constraint-heavy problems.

| Domain | Techniques |
|---|---|
| Dynamic Programming | State design, memoization, tabulation, space optimization |
| Graphs | BFS / DFS, Dijkstra, topological sort, Union-Find |
| Arrays & Strings | Sliding window, two pointers, prefix sums |
| Trees | Recursive decomposition, iterative traversal, LCA |
| Greedy & Backtracking | Interval scheduling, permutations, constraint pruning |

The goal is not just to solve — but to understand *why* a solution works, and where it breaks.
